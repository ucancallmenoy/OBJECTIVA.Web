What is Method Overriding?

Method overriding occurs when a subclass provides a specific implementation for a method that is already defined in its superclass.

The signature of the overridden method in the subclass must match the one in the superclass, and it allows the subclass to modify the behavior of the method as required.

Overriding vs. Overloading

Method Overriding and Method Overloading are both ways to define methods in Java, but they have distinct differences.

  • Method Overloading: Multiple methods with the same name but different parameters in the same class.
  • Method Overriding: A subclass provides a specific implementation for a method that is already defined in its superclass.

    • The {{'@Override'}} annotation

    The {{'@Override'}} annotation is used to indicate that a method in a subclass is intended to override a method in its superclass. While this annotation is not mandatory, it helps with readability and also allows the compiler to check that you are correctly overriding a method.

    
class Parent {{'{'}}
    void display() {{'{'}}
        System.out.println("Parent's display()");
    {{'}'}}
{{'}'}}

class Child extends Parent {{'{'}}
    {{'@Override'}}
    void display() {{'{'}}
        System.out.println("Child's display()");
    {{'}'}}
{{'}'}}

    Using 'super' in inheritance

    The super keyword is used to refer to the immediate parent class. It can be used to call methods, constructors, and access fields from the superclass.

    
class Parent {{'{'}}
    void display() {{'{'}}
        System.out.println("Parent's display()");
    {{'}'}}
{{'}'}}

class Child extends Parent {{'{'}}
    void display() {{'{'}}
        super.display();
        System.out.println("Child's display()");
    {{'}'}}
{{'}'}}

public class Main {{'{'}}
    public static void main(String[] args) {{'{'}}
        Child child = new Child();
        child.display();   
        // Output: Parent's display()
        // Output: Child's display()
    {{'}'}}
{{'}'}}
