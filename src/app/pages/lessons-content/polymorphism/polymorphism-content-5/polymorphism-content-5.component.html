Using Interfaces for Polymorphism

An interface is like a contract that says, "any class that implements this interface must have certain methods." When different classes implement the same interface, they can provide their own version of the methods. This lets you use the same interface to call different methods from different classes.

It's like different appliances (classes) having a power button (interface method), but each appliance operates differently when turned on.

To implemet it, follow this:


interface Appliance {{'{'}}  
    void turnOn(); // All appliances must implement this method  
{{'}'}}  

class Fan implements Appliance {{'{'}}  
    {{'@Override'}} 
    public void turnOn() {{'{'}}  
        System.out.println("Fan is spinning");  
    {{'}'}}  
{{'}'}}  

class Light implements Appliance {{'{'}}  
    {{'@Override'}}
    public void turnOn() {{'{'}}  
        System.out.println("Light is glowing");  
    {{'}'}}  
{{'}'}}  

public class Main {{'{'}}  
    public static void main(String[] args) {{'{'}}  
        Appliance appliance1 = new Fan(); // Polymorphism: Appliance reference  
        Appliance appliance2 = new Light();  

        appliance1.turnOn(); // Output: Fan is spinning  
        appliance2.turnOn(); // Output: Light is glowing  
    {{'}'}}  
{{'}'}}

Achieving Loose Coupling with Interfaces

Loose coupling means your code is flexible and doesn't depend too much on specific details. Using interfaces helps because your classes only need to know about the interface, not the actual class doing the work.

It's like a universal charger that can charge different devices (phone, tablet, etc.) because they all follow the same charging protocol (interface).

How to implement it?


interface Charger {{'{'}}  
    void charge(); // Common method for charging  
{{'}'}}  

class Phone implements Charger {{'{'}}  
    {{'@Override'}}
    public void charge() {{'{'}}  
        System.out.println("Phone is charging");  
    {{'}'}}  
{{'}'}}  

class Tablet implements Charger {{'{'}}  
    {{'@Override'}}
    public void charge() {{'{'}}  
        System.out.println("Tablet is charging");  
    {{'}'}}  
{{'}'}}  

public class Main {{'{'}}  
    public static void main(String[] args) {{'{'}}  
        Charger device = new Phone(); // Loose coupling: Easy to swap with other classes  
        device.charge(); // Output: Phone is charging  

        device = new Tablet();  
        device.charge(); // Output: Tablet is charging  
    {{'}'}}  
{{'}'}}

Interface-Based Polymorphism

With interfaces, you can treat different objects the same way. Even though they are different classes, they can still be treated as the same type (the interface).

It's like hiring workers (interface) to do a task, where the worker could be a painter, plumber, or electrician (different implementations).

How to implement it?


interface Worker {{'{'}}  
    void doWork(); // Define the task  
{{'}'}}  

class Painter implements Worker {{'{'}}  
    {{'@Override'}}
    public void doWork() {{'{'}}  
        System.out.println("Painter is painting the wall");  
    {{'}'}}  
{{'}'}}  

class Electrician implements Worker {{'{'}}  
    {{'@Override'}}
    public void doWork() {{'{'}}  
        System.out.println("Electrician is fixing the wiring");  
    {{'}'}}  
{{'}'}}  

public class Main {{'{'}}  
    public static void main(String[] args) {{'{'}}  
        Worker worker = new Painter(); // Interface-based polymorphism  
        worker.doWork(); // Output: Painter is painting the wall  

        worker = new Electrician();  
        worker.doWork(); // Output: Electrician is fixing the wiring  
    {{'}'}}  
{{'}'}}
